What can be done

Manage Minecraft from profile creation to shutdown.

BlockBaseMC is focused on the full Java Edition loop: create an instance, choose a version and loader, sign in, install content, launch, watch logs, and stop the process cleanly when you are finished.

Profiles

Each profile can have its own Minecraft version, loader, RAM values, JVM arguments, launch args, resolution, fullscreen setting, quick join server, and isolated game folder.

Accounts

Microsoft Java login uses the system browser. Saved sessions can reauth with refresh tokens, and offline/dev accounts are available for local testing.

Launch

The launcher downloads version metadata, assets, libraries, natives, and loader metadata, then starts Minecraft with profile-specific settings and live logs.

Running Instances

Only live Java processes are shown. Each entry is labeled by profile and account, includes the PID, and can be killed from the launcher.

Imports

Import ZIP modpacks, Modrinth packs, CurseForge packs, Prism/MultiMC instances, and existing .minecraft folders into BlockBaseMC profiles.

Content Manager

Installed mods, resource packs, and shader packs can be listed, disabled, restored, deleted, or opened in Explorer from the selected profile.

Mods & Packs

Search and install from real mod platforms.

Modrinth search works without a key. CurseForge support uses a user API key, then searches, installs, and attempts dependency downloads where project metadata allows it.

  • Mods install into the selected profile's mods folder.
  • Resource packs and shaders go to their matching folders.
  • Modpacks can create new profiles with the expected overrides.
  • Wrong-version mods are still Minecraft's job to reject, but logs make the reason readable.
