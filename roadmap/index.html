Ideas and next steps

Make the launcher easier, safer, and more complete over time.

BlockBaseMC is still early. The current goal is to make Java Edition reliable first, then improve compatibility, content installs, updates, and cross-platform builds.

Working now

Java launch path

Vanilla, snapshots, Fabric, Forge, Quilt, and NeoForge paths exist where upstream metadata can be resolved.

Working now

Account flow

Browser-based Microsoft login, saved account cards, refresh-token reauth, and offline/dev accounts.

Next

Cleaner mod installs

Better dependency resolution, clearer incompatible mod warnings, and update-all flows for selected profiles.

Next

Release workflow

Use GitHub Releases for public builds, cleaner changelogs, and a download button that always points to the newest exe.

Planned

macOS and Linux

The architecture avoids Windows-only assumptions where possible, but cross-platform builds still need testing and packaging.

Planned

Better repair tools

One-click repairs for assets, libraries, loader files, mod folders, and corrupted imports.

Known direction

Java Edition stays the focus.

Bedrock support was removed so the launcher can get one job right first: launching Minecraft Java with accounts, mods, profiles, and imports that actually work.

  • More reliable Forge, NeoForge, and Quilt metadata handling.
  • Better UI messages when Minecraft rejects a mod or version.
  • Profile export and import polish.
  • Theme controls and layout cleanup as the app grows.
