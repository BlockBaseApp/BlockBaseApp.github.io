Update history

From blank launcher shell to a working BlockBaseMC build.

This page tracks the major changes from the first local builds through the current release-fast64 build. Early builds moved quickly, so some entries group related work by milestone instead of pretending every test build was a public release.

Latest

release-fast64 - Pre-release updater fix

Fixed update checks for early GitHub pre-releases.

  • Updater now checks the GitHub Releases list instead of /releases/latest.
  • Pre-releases like v0.1.2 are now detected.
  • Downloads page now links to the Releases page.
  • Bumped the app version to 0.1.3.
Instance fix

release-fast63 - Instance creation fix

Fixed the Prism-style Library create flow so Create instance and Update selected actually run.

  • Fixed empty data-* button attributes being treated as false.
  • Create instance now confirms the profile was created.
  • New instances get unique folders even when names are reused.
  • Bumped the app version to 0.1.2.
Updater

release-fast62 - GitHub release update checker

Added a safe portable-app update flow that checks GitHub Releases instead of trying to replace the running exe.

  • Bumped the app version to 0.1.1 so future releases can be compared properly.
  • Checks itz-me-fisherYT/BlockBase-Launcher for the latest GitHub Release.
  • Compares the latest release version against the current launcher version.
  • Shows an update banner with View release, Download update, and Dismiss actions.
  • Added a manual Check updates button on the Downloads page.
Auth fix

release-fast60 - Microsoft silent reauth fix

Fixed the Microsoft refresh request for the default Minecraft client login flow by sending the required Xbox Live scope.

  • Silent reauth now uses the saved refresh token correctly.
  • Reauth should refresh without opening browser login.
  • Discord webhook update sent for the build.
Account UI

release-fast59 - Reauth button cleanup

Made the account card stop showing Reauth when silent reauth is not available.

  • Old Microsoft accounts show Silent reauth unavailable.
  • Added Upgrade login for accounts that need a one-time fresh sign-in.
  • Cleaned backend reauth messages so they are less scary.
Account UI

release-fast58 - Silent-only reauth

Changed Reauth so it never opens the Microsoft login flow as a fallback.

  • Reauth now succeeds silently or shows a message.
  • Removed the full browser-login fallback from the Reauth path.
Accounts

release-fast57 - Refresh-token reauth backend

Added the first real Microsoft reauth backend path.

  • Saved Microsoft refresh tokens for new logins.
  • Added a dedicated reauth IPC method.
  • Launch auth can attempt token refresh before falling back to offline/dev mode.
Branding

release-fast56 - BlockBaseMC rename

Renamed the launcher from BlockForge to BlockBaseMC.

  • Updated the app title, product name, docs, website copy, and launcher branding.
  • Changed new default profile folders to use .blockbasemc.
  • Kept internal storage keys stable so existing settings did not disappear.
Website

Website updates - BlockBaseApp.github.io

Moved the public website into the separate BlockBaseApp GitHub Pages repo.

  • Created a clean root website at https://blockbaseapp.github.io/.
  • Expanded the site into Home, Features, Built With, Roadmap, Updates, and Credits pages.
  • Added clean URLs like /features/ and fixed the shared CSS path.
  • Pointed download and source links to itz-me-fisherYT/BlockBase-Launcher.
Public repo

Source cleanup and repo split

Prepared the source repository for GitHub and separated the launcher from the website.

  • Removed old local release build folders from source tracking.
  • Ignored future release-fast* folders.
  • Kept launcher code on the main GitHub account and website code on the BlockBaseApp account.
Builds 52-55

release-fast52 to release-fast55 - Mods, imports, and Java focus

These builds focused on making the launcher useful for real Minecraft Java profiles.

  • Removed Bedrock support from the launcher scope.
  • Improved Running Instances so only live Java processes appear.
  • Added process kill controls labeled by profile/account.
  • Improved Modrinth and CurseForge browsing and installs for the selected library profile.
  • Added import flows for ZIP packs, .minecraft folders, Prism/MultiMC, Modrinth, and CurseForge packs.
  • Added the Open folder button next to Game folder.
Builds 48-51

release-fast48 to release-fast51 - Version and loader work

These builds focused on Minecraft version selection and loader launch paths.

  • Added a Minecraft version dropdown.
  • Started supporting Fabric, Forge, NeoForge, Quilt, Vanilla, and snapshots where metadata was available.
  • Fixed Fabric duplicate ASM classpath issues.
  • Improved handling for imported profiles that need a newer Minecraft version.
  • Added theme/color controls and fixed reset behavior.
Builds 44-47

release-fast44 to release-fast47 - Accounts, servers, and browsing

These builds focused on account management, server cards, and mod browser stability.

  • Added Microsoft Java account cards with username, email/status, skin actions, and delete controls.
  • Added 3D skin modal work and skin preview controls.
  • Added server creation, delete controls, quick join account selection, and server status details.
  • Fixed button clicks unexpectedly switching back to the Library tab.
  • Improved Browse and Mods & Packs tab freezing/search behavior.
Builds 42-43

release-fast42 to release-fast43 - First playable launcher builds

The earliest local builds turned the blank Electron shell into a usable Minecraft launcher interface.

  • Built the Prism-inspired sidebar layout and instance library.
  • Added profile creation, editing, duplicate, delete, RAM settings, resolution, JVM args, and launch args.
  • Made the Instances page scrollable and fixed resize/clipping issues.
  • Added Minecraft Java launch preparation, asset/library downloads, Java detection, and launch logs.
  • Fixed the quick play crash caused by passing more than one quick join option.
Before fast42

Initial prototype - from blank window to launcher shell

The first prototype was a plain Electron window before the custom launcher UI existed.

  • Started with a Windows desktop .exe target.
  • Created the core Electron app structure.
  • Planned the full Java/Bedrock feature list, then narrowed the real app to Java Edition first.
